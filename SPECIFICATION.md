# OpenNotes Format 
Version 1.0.0
## 1. Introduction
OpenNotes Format exists to be an open, not locked-in data format for handwritten note taking across several devices.
It should be the backbone of OpenSource Notetaking apps for multiple devices and operating systems. It's core value is that the data belongs the user. 
The format stores things like strokes, texts and images. It is designed to be fast and be small in space. Software that uses this format should be able to straightforward parse this format through various versions.
### 1.1 Purpose
The OpenNotes Format is a platform-independent file format for storing, reading, and modifying handwritten notes across multiple devices and operating systems.
### 1.2 Scope
This specifications defines the file syntax and how it is handled and interpreted. It does not define the storage, transportation or software to use it.
### 1.3 Design goals
The important goals for this format are:
- Efficient parsing
- Compact file size
- Cross-platform interoperability
- Long-term stability
- Forward compatibility
- Open specification
### 1.4 Non-Goals
This are not the goals of this format:
- Dependence on proprietary software
- Vendor lock-in
- Application-specific extensions
### 1.5 Format Principles
The OpenNotes Format is based on the following principles:
- User ownership of data
- Open and publicly documented specification
- Cross-platform compatibility
- Long-term accessibility
- Backward compatibility whenever practical
